## __ЧАСТЬ ПЕРВАЯ.__

## Знакомство с контролем версий. 
***

### __Что это и для чего может быть нужно?__

* Возможность хранить различные версии проекта.
* Возможность возвращаться к различным версиям проекта. 

GIT - это программа, которая берёт на себя вопросы контроля версий над проектом.

GIT - сохраняет в памяти не файлы целиком, а **РАЗНИЦУ** между файлами.
***
### НАЧАЛО РАБОТЫ

Итак, чтобы начать отслеживать изменения в нашем проекте, нам необходимо где-то хранить все файлы. Поэтому:
1. Создаём __*ПАПКУ*__ и даём ей __*ИМЯ*__ - там будет храниться вся наша работа. 
2. Запускаем программу __*«Visual Studio Code»*__.
3. Затем, в левом верхнем углу находим __*ПРОВОДНИК*__ (два квадратика с одним загнутым кончиком), нажимаем на него.
4. После этого нажимаем на строку __*«ОТКРЫТЬ ПАПКУ».*__
Находим **_"НУЖНУЮ ПАПКУ"_** и нажимаем __*«ОТКРЫТЬ».*__
5. Запускаем __*ТЕРМИНАЛ»*__
* в __*МЕНЮ*__ пункт __*ВИД*__ выбираем __*ТЕРМИНАЛ*__ и нажимаем на него.
 
Сначала проверим, что «GIT» у нас настроен, т.е. мы сейчас попросим его вывести свою версию на экран. 
И если у нас «GIT» установлен и настроен правильно, то мы должны увидеть на экране текущую версию этой программы. Для этого пишем в __*ТЕРМИНАЛЕ:*__
+ __git —version__ 
+ Нажимаем Enter

Теперь необходимо его __*ИНИЦИАЛИЗИРОВАТЬ*__ наш _ЛОКАЛЬНЫЙ РЕПОЗИТОРИЙ:_
+ __git init__
+ Нажимаем Enter

Чтобы узнать текущий __*СТАТУС*__, вызываем команду
+ __git status__
+ Нажимаем Enter

Далее, создаём новый __*ФАЙЛ:*__
1. Нажимаем на квадратик с загнутым кончиком, в левом окне.
2. Пишем название _ФАЙЛА_ (на английском языке), ставим _ТОЧКУ_ и _РАСШИРЕНИЕ._

> _Примечание: Файлы можно сделать с любым расширением (.md, .txt, .doc и т.д.), какое вам будет нужно. При этом помним, что после названия ФАЙЛА, перед любым РАСШИРЕНИЕМ ставится ТОЧКА)._

Далее, в _ОКНЕ для ввода текста_, которое находится выше _ТЕРМИНАЛА_, пишем _ТЕКСТ_ и _СОХРАНЯЕМ_ его. 
> _Примечание: Для сохранения текста, чтобы постоянно не нажимать на (__cmd s__) в МЕНЮ, пункт ФАЙЛ, выбираем АВТОСОХРАННИЕ (ставим галочку)._

Теперь попросим «GIT» указать текущий статус. 
Пишем в _ТЕРМИНАЛЕ_ команду:
+ __git status__
+ Нажимаем Enter

Видим неотслеживаемый файл (КРАСНАЯ строчка).

> _Примечание: Если на клавиатуре нажимать стрелки "вверх" (↑) или "вниз" (↓), то  можно выбрать команду из тех, которые мы писали._

Для того, чтобы «GIT» начал отслеживать изменения в нашем файле, необходимо их ДОБАВИТЬ:
+ __git add «имя файла с точкой и расширением»__
+ Нажимаем Enter

Если никаких сообщений об ошибках нет, значит всё прошло успешно.

> _Примечание: Чтобы не писать имя файла полностью, достаточно набрать ДВЕ буквы НАЗВАНИЯ ФАЙЛА и нажать на кнопку Tab (**→|**), которая располагается в крайнем левом столбце клавиатуры, третья снизу, под клавишей «Esc» и
файл ДОПИШЕТСЯ сам._

Теперь посмотрим _СТАТУС_. Ищем команду, нажимая стрелки (↑) или (↓) и выбираем:
+ __git status__ 
+ Нажимаем Enter

Видим что «GIT» начал отслеживать наш файл (ЗЕЛЁНАЯ строка).

Далее СОХРАНЯЕМ (фиксируем) текущий статус:
+ __git commit -m «Сюда (в кавычках) пишем комментарий о том, что мы сделали»__
+ Нажимаем Enter

Видим сообщение с текстом, что произошло.

> _Примечание: Принято писать комментарий на АНГЛИЙСКОМ языке. Но, если вы не знаете английского языка, можно (для начинающих), для удобства, писать на РУССКОМ языке._

Давайте как-то изменим файл. Напишем в текстовом окне _"КАКОЙ-ТО ТЕКСТ"_, чтобы файл видоизменился.

Затем, смотрим на текущий СТАТУС. Ищем команду, нажимая (↑) или (↓) и выбираем:

+ __git status__
+ Нажимаем Enter

Видим, некоторые сообщения от гита.
* Он говорит, что мы находимся на ветке МАСТЕР (о "Ветках" чуть позже).
+ Он говорит, что у нас есть некоторые изменения, которые пока ещё не сохранены (КРАСНАЯ строка).
+ И показывает, что у нас изменён файл.

Поэтому, пишем команду для ДОБАВЛЕНИЯ:
+ __git add «имя файла с точкой и расширением»__
+ Нажимаем Enter

Далее ещё раз смотрим на СТАТУС:
+ __git status__
+ Нажимаем Enter

Появляется сообщение и (ЗЕЛЁНАЯ строка).

Далее СОХРАНЯЕМ (фиксируем) текущий статус:
+ __git commit -m «Сюда (в кавычках) пишем комментарий о том, что мы сделали»__
+ Нажимаем Enter

Видим сообщение с текстом, что произошло.

Теперь у нас есть ДВА СОХРАНЕНИЯ, с которыми можем работать. Мы можем переходить от одной версии файла к другой, чтобы понять какие версии существуют. Для этого есть специальная команда - _ЖУРНАЛ ВЕРСИЙ:_
+ __git log__
+ Нажимаем Enter

Появится сообщение о том, что:
+ Есть некоторая фиксация, т.е. некоторые сохранения (ЖЁЛТАЯ строка с цифрами и символами).
+ И у каждого из этих сохранений есть свой текстовый комментарий (тот, который мы оставили).

> _НИЖНЕЕ (оно считается - первым) сохранение (ЖЁЛТАЯ строка с цифрами и символами) и с КОММЕНТАРИЕМ относится к тому, когда мы создали новый файл._
> _ВЕРХНЕЕ (оно считается - вторым) сохранение (ЖЁЛТАЯ строка с цифрами и символами) и с КОММЕНТАРИЕМ относится к тому, когда мы добавляли новую строку и то, что мы сейчас находимся на ней (HEAD → master) ЦВЕТНАЯ строка._

Если мы хотим перейти к предыдущей версии этого файла, вызываем команду git, пробел, checkout, пробел, вставляем скопированные первые ЧЕТЫРЕ цифры/символа, в данном случае, НИЖНЕЙ (первой строки ЖЁЛТОГО цвета):

>_ЧЕТЫРЕ символа будет достаточно для того, чтобы понять, какое сохранение необходимо запустить)_

+ __git checkout и четыре скопированных цифр/символа__
+ Нажимаем Enter

Видим, что иcчезла ВТОРАЯ строка, которую мы писали. 

Теперь мы можем вернуться ОБРАТНО. Копируем ЧЕТЫРЕ цифры/символа ВЕРХНЕЙ (второй строки ЖЁЛТОГО цвета):
+ __git checkout и четыре скопированных цифр/символа__
+ Нажимаем Enter

Видим, что мы перешли обратно к НОВОЙ (актуальной) версии. И у  нас опять появилась ВТОРАЯ строка, которую мы писали. 

По сути мы прошли весь путь СОЗДАНИЯ ФАЙЛА и ФИКСАЦИИ изменения в нём.

> _ВАЖНО ПОМНИТЬ:
что после того, как мы попереключались между разными версиями, нам необходимо вернуться в самое актуальное состояние._

Для этого необходимо вызвать команду:
+ __git checkout master__
+ Нажимаем Enter

## _Изучим ветвление и переход между ветвями_
***
### В прошлый раз мы научились сохранять правки (git commit).

Повторим команды:
* __git --version__ - проверить текущую установленную версию программы;
* __git init__ — создать репозиторий в папке на локальной машине; 
* __git status__ - посмотреть текущее состояние гита, есть ли изменения, которые нужно закоммитить (сохранить);
* __git add + название файла с точкой и расширением__ — отслеживать добавленные файлы, для последующего коммита;
* __git commit + "комментарий в ковычках"__ — сохранить текущее состояние;
* __git log__ — вызывать список (журнал) действий и сохранений;
* __git chechout__ — переключиться между разными версиями (разными коммитами);
* __git diff__ — показать разницу между версиями.
***
### Начало работы.
Ветви бывают:
+ Чистовик -это главная ветка. Сюда мы будем перемещать всё то, что сделаем в черновиках. Допустим эта ветка __"master"__
+ Черновики - это ветки, которые мы будет создавать и удалять по необходимости. Для примера возьмём ТРИ ветки - __"text_1", "text_2", "text_3".__

Для этого:

1. Создаём файл с точкой и расширением;
2. Открываем Терминал;
3. Инициализируем наш локальный репозиторий;
4. Смотрим статус;
5. Добавляем файл;
6. Коммитим (фиксируем) его;

### Зададим структуру файла.


7. Создадим набор заголовков в текстовом окне:
+ Решётка (#), пробел, название _ЗАГОЛОВОКА_ (Заголовок должен быть один)
+ Две решётки (##), пробел, название _ПОДЗАГОЛОВКОВ_ (Подзаголовков может быть несколько)

> _Примечание: Можно разделить заголовки и подзаголовки линиями. Для этого:_
>+ _Ставим ТРИ ЗВЁЗДОЧКИ (***)_
>+ _Переходим на новую строку, пишем название ЗАГОЛОВОКА или ПОДЗАГОЛОВКА_
>+ _Ставим ТРИ ЗВЁЗДОЧКИ (***)_
>+ _С новой строки начинаем писать текст_

9. Сохраним изменения в нашем файле:
+ __git status__ 
+ Нажимаем Enter
+ __git add «имя файла с точкой и расширением»__ 
+ Нажимаем Enter
+ __git status__ 
+ Нажимаем Enter
+ __git commit -m «Сюда (в кавычках) пишем комментарий о том, что мы сделали»__ 
+ Нажимаем Enter

### Создаём ветки.

Ветка с английского - __branch__. 

13. Чтобы вывести на экран сколько ВЕТОК у нас есть, зададим команду:

+ __git branch__ 
+ Нажимаем Enter

И видим, что у нас пока одна ветка с названием __master__ (т.е. ЗВЁЗДОЧКА (*), которая показывает на какой ветке мы находимся и название ветки master (ЗЕЛЁНЫМ цветом).


> _Примечание: Чтобы очистить поле (спрятать всё то, что было) в ТЕРМИНАЛЕ, пишем:_
>+ __clear__ 
>+ _Нажимаем Enter и работаем дальше._

14. Чтобы создать новую ветку, нужно набрать команду git branch, пробел и название ветки (любое) на английском языке, например, __"text_1"__ и получится так:

+ __git branch text_1__ 
+ Нажимаем Enter

15. Чтобы проверить, создана ли у нас ветка, вызываем команду:

+ __git branch__ 
+ Нажимаем Enter

16. Чтобы перейти на новую  ветку, пишем:

+ __git checkout text_1__ 
+ Нажимаем Enter

17. Чтобы проверить на новой ли ветке мы находимся, вызываем команду:

+ __git branch__ 
+ Нажимаем Enter

> _Примечание: Можно создать ветку так, чтобы она сразу перешла в статус АКТИВНОЙ, т.е. как бы соединяем пункты 14), 15), 16) в одну команду:_
>+ __git checkout -b text_1__
>+ Нажимаем Enter

> Чтобы убедиться, что мы всё сделали правильно, вызываем команду:
>+ __git branch__
>+ Нажимаем Enter

> И смотрим результат.

### Перемещение веток.

После того, как мы создали структуру из нескольких _ПОДЗАГОЛОВКОВ_, например из ТРЁХ. 

Создали новую ПЕРВУЮ ветку (черновик), например __text_1__.

Нам необходимо перейти на ГЛАВНУЮ ветку master (чистовик), для этого пишем команду:
+ __git checkout master__
+ Нажимаем Enter

И оттуда создаём новую ВТОРУЮ ветку (черновик), например text_2.

Затем опять переходим на ГЛАВНУЮ ветку master (чистовик).

И оттуда создаём новую ТРЕТЬЮ ветку (черновик), например, text_3.

Мы можем перемещаться между ветками для написания нужного текста, корректировки, изменений, следуя правилам:

1. Вначале смотрим, на какой ветке находимся в данный момент, набираем команду:
+ __git branch__ 
+ Нажимаем Enter
2. Затем, выбираем ту ветку, на которую хотим перейти, например, ветку text_1:
+ __git checkout text_1__
+ Нажимаем Enter
3. Затем убеждаемся, что мы находимся именно на этой ветке, набираем команду:
+ __git branch__ 
+ Нажимаем Enter

Смотрим результат.

### Заливаем в master (чистовик).

После того, как мы оформили одну, несколько или все ветки - их можно СЛИТЬ в чистовик. Например, возьмём ветку text_1 (черновик). 

Для этого сначала смотрим, на какой ВЕТКЕ мы сейчас находимся:
+ __git branch__
+ Нажимаем Enter

Если мы уже находимся на ветке text_1 (черновик), то смотрим на СТАТУС, всё ли у нас сохранено, чтобы перелить ветку  (черновик) в ветку master (чистовик):
+ __git status__
+ Нажимаем Enter

После этого мы возвращаемся на ГЛАВНУЮ ветку master (чистовик):
+ __git checkout master__
+ Нажимаем Enter

Далее, чтобы перенести ветку (черновик) в ветку (чистовик) нам нужна команда git merge - СЛИЯНИЕ и название ветки (черновика), т.е. пишем:
+ __git merge next_1__
+ Нажимаем Enter

Видим, что в нашей ветке master (чистовик), файл обновился.

Далее смотрим журнал изменений:
+ __git log__
+ Нажимаем Enter

Смотрим на верхнюю строчку, она считается последней (актуальной) в журнале "log", где показано в скобках с СИНИМ и ЗЕЛЁНЫМ цветом  (HEAD → master, text_1):
+ Что мы находимся на ветке master,
+ Что у нас также есть ветка text_1, которая находится именно на этом же КОММИТЕ
+ И мы видим все изменения, которые совершали, т.е. строчки с цифрами и символами (ЖЁЛТОГО цвета) и КОММЕНТАРИИ к ним.
> _Примечание: Если на последней строке в ТЕРМИНАЛЕ не получается писать команды, т.е. там показаны двоеточие (:) и белый "флажок"(начало для ввода команд), следует нажать на английскую букву кью и продолжить работу:_
>+ Q, q (_кью_)

Теперь можно очистить Терминал командой:
+ __clear__
+ _Нажимаем Enter_

### Удаление веток.

Теперь мы можем удалить ветку text_1 (черновик). 

Для этого набираем команду git branch, пробел, -d (delete - удалить), пробел и text_1 (имя ветки, которую хотим удалить):
+ __git branch -d text_1__
+ Нажимаем Enter

Смотрим на список веток:
+ __git branch__
+ Нажимаем Enter

И видим, что ветки text_1 нет в нашем списке. Она удалена.

 ### Разграничение понятий Git и GitHub.

__Git__ - это ПРОГРАММА, которая устанавливается на ваш компьютер и внутри него, локально, выполняет те команды, которые, вы хотите, чтобы она выполнила.

+ __git__ — одна из систем контроля версий
+ Способ организации и поддержания версионности
+ Самая популярная система контроля версий

__GitHub__ - это СЕРВИС: 
+ Предоставляет компания __Microsoft__
+ Самый популярный сервис __Git__
+ Много полезных функций
+ Огромный архив различного
кода

Сервис __GitHub__ позволяет интегрироваться с программой __Git__ и настроить удалённую работу с вашим локальным репозиторием, т.е:

+ Вы можете работать ЛОКАЛЬНО в программе __Git__
+ А можете хранить свой репозиторий на __GitHub__.
Всё это можно делать с помощью команд, которые есть в программе __Git__.

## Инструкция создания РЕПОЗИТОРИЯ (второй способ)

Первый способ у нас был через __git init__, когда мы создавали свой РЕПОЗИТОРИЙ с нуля

Теперь рассмотрим второй способ создания РЕПОЗИТОРИЕВ, через __git clone__, т.е. когда мы можем взять чей-то чужой РЕПОЗИТОРИЙ, который, кстати тоже был когда-то создан через __git init__ и продолжить работать с этим РЕПОЗИТОРИЕМ уже локально у себя.
Итак, для начала мы:

1. Создаём __*ПАПКУ*__ и даём ей имя, например __lesson__ - там будет храниться скопированный репозиторий. 
2. Запускаем программу __*«Visual Studio Code»*__ (__VSCode__).
3. Затем, в левом верхнем углу находим __*ПРОВОДНИК*__ (два квадратика с одним загнутым кончиком), нажимаем на него.
4. После этого нажимаем на строку __*«ОТКРЫТЬ ПАПКУ».*__
Находим папку __lesson__ и нажимаем __*«ОТКРЫТЬ».*__

Здесь мы не будем создавать свой локальный репозиторий, а возьмём чей-то уже готовый.

И для того, чтобы убедиться, что у нас __Git__ не настроен, вызовем команду:
+ __git status__

### Команда pull request

В этом случае должен быть создан один основной аккаунт. Один основной Репозиторий.

Доступ к этому Репозиторию есть только у ограниченного числа лиц.

Но многие другие люди могут тоже поучаствовать в вашем проекте.

в __Git__ это называется __pull request__
+ ➜ команда для предложения изменений.
+ ➜ запрос на вливание изменений в репозиторий.
+ В больших компаниях один ответственный за проект создает аккаунт.
+ Другие пользователи дают команду pull request.
+ Предлагать изменения на GitHub нужно в отдельной ветке.

Сначала пользователь копирует репозиторий на свой компьютер, делает fork репозитория, затем клонирует версию на своём ПК, создаёт ветку с предлагаемыми изменениями, отправляет изменения командой push в свой аккаунт на GitHub и даёт команду pull request.

Как сделать pull request?
