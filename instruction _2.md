 ### Разграничение понятий Git и GitHub.

__Git__ - это ПРОГРАММА, которая устанавливается на ваш компьютер и внутри него, локально, выполняет те команды, которые, вы хотите, чтобы она выполнила.

+ __git__ — одна из систем контроля версий
+ Способ организации и поддержания версионности
+ Самая популярная система контроля версий

__GitHub__ - это СЕРВИС: 
+ Предоставляет компания __Microsoft__
+ Самый популярный сервис __Git__
+ Много полезных функций
+ Огромный архив различного
кода

Сервис __GitHub__ позволяет интегрироваться с программой __Git__ и настроить удалённую работу с вашим локальным репозиторием, т.е:

+ Вы можете работать ЛОКАЛЬНО в программе __Git__
+ А можете хранить свой репозиторий на __GitHub__.
Всё это можно делать с помощью команд, которые есть в программе __Git__.

## Инструкция создания РЕПОЗИТОРИЯ (второй способ)

Первый способ у нас был через __git init__, когда мы создавали свой РЕПОЗИТОРИЙ с нуля

Теперь рассмотрим второй способ создания РЕПОЗИТОРИЕВ, через __git clone__, т.е. когда мы можем взять чей-то чужой РЕПОЗИТОРИЙ, который, кстати тоже был когда-то создан через __git init__ и продолжить работать с этим РЕПОЗИТОРИЕМ уже локально у себя.
Итак, для начала мы:

1. Создаём __*ПАПКУ*__ и даём ей имя, например __lesson__ - там будет храниться скопированный репозиторий. 
2. Запускаем программу __*«Visual Studio Code»*__ (__VSCode__).
3. Затем, в левом верхнем углу находим __*ПРОВОДНИК*__ (два квадратика с одним загнутым кончиком), нажимаем на него.
4. После этого нажимаем на строку __*«ОТКРЫТЬ ПАПКУ».*__
Находим папку __lesson__ и нажимаем __*«ОТКРЫТЬ».*__

Здесь мы не будем создавать свой локальный репозиторий, а возьмём чей-то уже готовый.

И для того, чтобы убедиться, что у нас __Git__ не настроен, вызовем команду:
+ __git status__
+ Нажимаем Enter

Должно появиться сообщение, что это не __git__ репозиторий, какая-то фатальная ошибка и наша программа отказывается дальше работать, т.е. так мы убедимся, что здесь пустая папка без каких-либо репозиториев.

Теперь необходимо перейти на сайт github.com и здесь через поиск вы можете находить разных авторов, разные проекты.

Мы возьмём один из готовых репозиториев, например (https://github.com/ilnar-geekbrains/version_control_lection_3), с которым будем работать (ЖМИ НА ССЫЛКУ, чтобы туда перейти).

После того, как мы перешли по этой ссылке на сайт  __GitHub__, здесь мы видим два файла (квадратики с загнутыми кончиками).

+ Аккаунт автора этого репозитория,
+ И указан компьютер, на котором автор работает.
+ А дальше указана папка, в которой мы с вами находимся.

Так вот, сейчас команда __git status__ запускается в папке __lesson__.

А в папке __lesson__ нет РЕПОЗИТОРИЯ.

+ Нам нужно будет указать всю первую строчку (т.е. в ней написано, что программа __"git"__ должна будет понять, что у нас появляется новый удалённый Репозиторий. И ещё здесь мы указываем ссылку (адрес) на тот удалённый Репозиторий, с которым нужно будет работать. И каждый раз, когда мы захотим, что-то отправить в интернет, __"git"__ будет знать, что отправлять ему нужно будет на этот адрес (ссылку). То есть, по сути, мы связываем наш ЛОКАЛЬНЫЙ Репозиторий с нашим УДАЛЁННЫМ Репозиторием.

+ И после этого уже направляем то, что у нас есть на ЛОКАЛЬНОМ Репозитории в интернет (push -u origin main).

Все ссылки мы можем всегда скопировать (нажать на квадратики с загнутым кончиком) и просто вставить в наш проект. И у нас всё прекрасно заработает.

### В обратном направлении.

Тоже самое у нас происходит и в обратном направлении.

Предполижим, что вы с одного компьютера написали этот текст, а на другом компьютере внесли какие-то изменения.

То есть вы точно также сделали ЛОКАЛЬНЫЙ Репозиторий на другом компьютере. Что-то в нём поменяли. Загрузили на __GitHub__.

И получится, что у вас на __GitHub__ версия более актуальная, чем на текущем компьютере.

Давайте мы с вами это сделаем, только сделаем не с другого компьютера, а прямо, напрямую в __GitHub__ изменим наш файл. 

Давайте добавим здесь информацию, например, Совершенно с этим согласен!

После этого, соответственно, мы можем сохранить эти изменения:
+ Для этого нужно опуститься в низ странички
+ Добавить в строке "Commit changes" свой коммит, например, Изменения с другого компьютера.
+ Предположим, что вы на другом своём компьютере, сделали локальный репозиторий, добавили там изменения и закоммитили его.
+ Затем нажимаем на ЗЕЛЁНУЮ кнопку "Commit changes".
+ Допустим, что вы после этого направили всё на __GitHub__
+ И вот у вас на __GitHub__ версия из трёх строчек, а на ЛОКАЛЬНОМ компьютере, с которого мы с вами, только что работали у нас всего лишь две строчки.

Так же на __GitHub__ (если нажать на кнопку "main", с левой сторонымы, то можем посмотреть на ИСТОРИЮ изменений, здесь есть также ветки, между которыми можно переключаться.

Также можно посмотреть на все коммиты, которые были сделаны, кнопка "History" находится с правой стороны. Если на неё нажать, откроется страничка с коммитами.

При этом обратите внимание, что если первые два коммита совершал один пользователь, то последний коммит совершил какой-то другой пользователь. 

У них разные аккаунты, разные люди делали, возможно с разных компьютеров.

Соответственно здесь, если мы вызовем команду:

То мы увидим всего лишь два коммита.

Что нам нужно сделать?
Нам теперь необходимо взять актуальную версию с __GitHub__ и сделать так, чтобы у нас на ЛОКАЛЬНОМ компьютере, наш ЛОКАЛЬНЫЙ Репозиторий был полностью синхронизирован с ним.

Это можно сделать с помощью команды pull - тянуть, т.е. стянуть с УДАЛЁННОГО Репозитория, который мы с вами создали.

> _Обратите внимание, что __"pull"__ является составной командой. Она не только подгрузит все изменения, но и попытается СМЁРЖИТЬ наши ветки, т.е. она попытается сделать "merge"(слияние) состояние, которое было на __GitHub__ с состоянием, которое у нас происходит ЛОКАЛЬНО._

Давайте запустим эту команду и посмотрим, что же у нас произойдёт:
+ __git pull__
+ Нажимаем Enter

И видим, что у нас появился текст "Совершенно с этим согласен", т.е. всё пришло на наш компьютер.

Взглянем на журнал изменений:
+ __git log__
+ Нажимаем Enter

__git log__ у нас теперь показывает следующее:
+ У нас есть одно изменение с другого компьютера
+ У нас есть два коммита с нашего ЛОКАЛЬНОГО компьютера.

> _Обратите внимение, что у нас также указаны два разных пользователя. Именно поэтому, когда вы начинаете пользоваться Гитом, вам необходимо представиться, чтобы __git__ знал, кто делает такие изменения. Если изменения делаются разными людьми, то вы всегда сможете узнать, а кто же сделал тот или иной коммит._

Когда вы работаете не один, а в команде - это становится очень-очень важно.

### Команда pull request

В этом случае должен быть создан один основной аккаунт. Один основной Репозиторий.

Доступ к этому Репозиторию есть только у ограниченного числа лиц.

Но многие другие люди могут тоже поучавствовать в вашем проекте.

в __Git__ это называется __pull request__
+ ➜ команда для предложения изменений.
+ ➜ запрос на вливание изменений в репозиторий.
+ В больших компаниях один ответственный за проект создает аккаунт.
+ Другие пользователи дают команду pull request.
+ Предлагать изменения на GitHub нужно в отдельной ветке.

Сначала пользователь копирует репозиторий на свой компьютер, делает fork репозитория, затем клонирует версию на своём ПК, создаёт ветку с предлагаемыми изменениями, отправляет изменения командой push в свой аккаунт на GitHub и даёт команду pull request.

Как сделать pull request?

+ Делаем fork (ответвление) репозитория

+ Делаем git clone СВОЕЙ версии репозитория 

+ Создаем новую ветку и в НЕЕ вносим свои изменения 

+ Фиксируем изменения (делаем коммиты)

+ Отправляем свою версию в свой GitHub

+ На сайте GitHub нажимаем кнопку pull request
